zsyLearnC gRPC 使用与关键代码流转

这份文档只讲你仓库里的真实代码路径：从 proto、到 CMake 生成代码、到 Server/Client 调用、到线程轮次协调。

项目: zsyLearnC 语言: C++17 通信: gRPC unary

目录

1. 程序目标 2. proto 契约 3. 编译期代码生成 4. 运行期总编排 5. 服务端处理链 6. 客户端发送链 7. 线程轮次协作 8. 第 1 轮完整时序 9. 核心知识点 10. 下一步追问模板

1. 程序目标

同一进程里启动两个 gRPC server，再用两个 sender 线程轮流互发消息，共 10 轮。

node1 sender -> gRPC -> node2 service -> ACK node2 sender -> gRPC -> node1 service -> ACK （每次发送后 sleep 5 秒，轮次由 TurnCoordinator 控制）

2. proto 契约（proto/pingpong.proto）

syntax = "proto3";
package pingpong;

message PingRequest {
  string from = 1;
  string to = 2;
  string body = 3;
  string sent_at = 4;
  int32 sequence = 5;
}

message PingReply {
  bool received = 1;
  string ack_from = 2;
  string ack_at = 3;
  string note = 4;
  int32 sequence = 5;
}

service Messenger {
  rpc SendMessage(PingRequest) returns (PingReply);
}

关键点：客户端和服务端都以这份契约生成类型和接口，业务代码只围绕这些类型开发。

3. 编译期代码生成（CMakeLists.txt）

find_program(GRPC_CPP_PLUGIN_EXECUTABLE grpc_cpp_plugin REQUIRED)

add_custom_command(
  OUTPUT "${PROTO_SRCS}" "${PROTO_HDRS}" "${GRPC_SRCS}" "${GRPC_HDRS}"
  COMMAND "${PROTOC_EXECUTABLE}"
  ARGS
    --proto_path "${CMAKE_CURRENT_SOURCE_DIR}/proto"
    --cpp_out "${GENERATED_DIR}"
    --grpc_out "${GENERATED_DIR}"
    --plugin=protoc-gen-grpc=${GRPC_CPP_PLUGIN_EXECUTABLE}
    "${PROTO_FILE}"
)

add_library(pingpong_proto "${PROTO_SRCS}" "${GRPC_SRCS}")

这段配置把 pingpong.pb.* 和 pingpong.grpc.pb.* 生成为可编译库，供业务层链接。

5.2 RPC 处理（src/common/messenger_service.cpp）

grpc::Status MessengerService::SendMessage(...){
  logger_.Log("... received sequence=...");
  reply->set_received(true);
  reply->set_ack_from(self_name_);
  reply->set_ack_at(now);
  reply->set_note("received");
  reply->set_sequence(request->sequence());
  return grpc::Status::OK;
}

OK 路径：收到消息 -> 写日志 -> 回 ACK 字段。

6. 客户端发送链（src/common/grpc_message_client.cpp）

stub_(pingpong::Messenger::NewStub(
  grpc::CreateChannel(target, grpc::InsecureChannelCredentials())));

bool GrpcMessageClient::Send(const std::string& from, const std::string& to, int sequence) {
  pingpong::PingRequest request;
  request.set_from(from);
  request.set_to(to);
  request.set_body("hello from " + from);
  request.set_sent_at(CurrentTimeString());
  request.set_sequence(sequence);

  grpc::ClientContext context;
  pingpong::PingReply reply;
  const grpc::Status status = stub_->SendMessage(&context, request, &reply);

  if (!status.ok()) {
    logger_.Log("... failed: " + status.error_message());
    return false;
  }
  logger_.Log("... ack=true ...");
  return true;
}

注意：这是同步 unary 调用。没有 deadline、没有重试，适合 demo。

7. 线程轮次协作（TurnCoordinator + Node1/Node2Sender）

7.1 轮次协调器

Turn TurnCoordinator::WaitFor(Turn expected_turn) {
  cv_.wait(lock, [&]() { return turn_ == expected_turn || turn_ == Turn::Done; });
  return turn_;
}

void TurnCoordinator::SetTurn(Turn next_turn) {
  turn_ = next_turn;
  cv_.notify_all();
}

7.2 Sender 逻辑（关键行）

// node1_sender.cpp
if (coordinator_.WaitFor(common::Turn::Node1ToNode2) == common::Turn::Done) return;
client_.Send("node1", "node2", round);
coordinator_.SetTurn(common::Turn::Node2ToNode1);

// node2_sender.cpp
if (coordinator_.WaitFor(common::Turn::Node2ToNode1) == common::Turn::Done) return;
client_.Send("node2", "node1", round);
if (round == rounds_) coordinator_.SetTurn(common::Turn::Done);
else coordinator_.SetTurn(common::Turn::Node1ToNode2);

8. 第 1 轮完整时序

[Node1Sender] WaitFor(Node1ToNode2) -> GrpcMessageClient::Send("node1","node2",1) -> stub->SendMessage(...) -> [node2 MessengerService::SendMessage] set_received(true), set_ack_from("node2"), return OK <- receive PingReply log: node1 -> node2 sequence=1 ack=true ... sleep(5s) SetTurn(Node2ToNode1) [Node2Sender] 开始下一半轮次

9. 你必须吃透的 9 个点

  1. .proto 是通信契约，客户端和服务端都依赖它。
  2. grpc_cpp_plugin 缺失会直接导致构建失败。
  3. Service 是服务端入口，Stub 是客户端入口。
  4. 这个项目使用同步 unary RPC，不是流式 RPC。
  5. grpc::Status 判定传输层成功失败；业务成功需看 reply 字段。
  6. 日志必须线程安全，否则多线程输出会串行污染。
  7. 轮次顺序由 TurnCoordinator 保证，不是 gRPC 自动保证。
  8. 当前是明文信道（InsecureChannelCredentials），生产需 TLS。
  9. 若要高吞吐，下一步是 async + CompletionQueue。
