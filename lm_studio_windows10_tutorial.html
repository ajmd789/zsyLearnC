  • 会安装并配置 LM Studio。

  • 会设置温度、上下文长度、采样等关键参数。

2. 安装前准备

2.1 最低建议配置

资源建议说明
系统Windows 10 x64建议更新到较新补丁版本
内存至少 16GB8GB 也能跑小模型，但体验有限
显卡NVIDIA 6GB+ 显存更好无独显可纯 CPU 跑，但速度慢
硬盘至少 40GB 空间模型文件很占空间

2.2 先做系统自检（PowerShell）

winver
wmic cpu get name
wmic computersystem get totalphysicalmemory
nvidia-smi
说明：
1) nvidia-smi 有输出说明驱动基本正常。
2) 没有 NVIDIA 卡也能跑，直接走 CPU 推理即可。

3. 安装 LM Studio

  1. 访问 LM Studio 官网并下载 Windows 安装包（建议从官网拿最新稳定版）。
  2. 双击安装，按默认选项完成。
  3. 首次启动时，允许应用通过防火墙（后续开本地 API 会用到）。
  4. 进入设置页，检查模型下载目录（建议放到大容量盘，如 D:\models\lmstudio）。

3.1 必做设置

  • 模型目录：改到空间充足的磁盘。
  • 自动更新：可开，但生产环境建议手动更新。
  • 本地服务默认端口：记住（通常是 1234）。

4. 选模型与量化（最关键）

4.1 先理解模型格式

  • GGUF：本地推理最常见格式，适合 LM Studio。
  • Instruct：聊天/问答场景优先选它。
  • Base：偏底座模型，不适合直接聊天。

4.2 量化怎么选（Q4 / Q5 / Q8）

量化速度质量内存占用建议
Q4快中低入门首选，先跑起来
Q5中较好中综合平衡
Q8慢更好高硬件充足再考虑

4.3 按内存选模型大小（经验值）

  • 8GB RAM：1B~3B 模型，Q4。
  • 16GB RAM：7B 模型，Q4/Q5。
  • 32GB RAM：14B 模型，Q4。
  • 64GB RAM：32B 模型可尝试（仍建议先 Q4）。
实战策略：先选 7B Instruct + Q4 跑通，再升级模型，不要一开始就冲大模型。

5. 第一次本地对话（完整步骤）

  1. 在 LM Studio 里打开模型发现/下载页面，下载一个 7B Instruct GGUF（Q4）。
  2. 下载完成后，进入聊天页加载模型。
  3. 确认模型状态是 “Loaded/Ready”。
  4. 发第一条消息测试： 
    请用 5 条要点解释什么是 gRPC，并给出一个 C++ 的最小调用流程。
  5. 如果能持续返回内容，说明本地推理链路已打通。

6. 推理参数讲透（别再盲调）

参数作用建议起点
Temperature越高越发散，越低越保守0.7
Top P控制采样范围0.9
Max Tokens最大输出长度512~2048
Context Length能记住的上下文窗口4k 或 8k 起步
Repeat Penalty抑制重复话术1.05~1.15

6.1 常见场景预设

  • 写代码：Temperature 0.2~0.5，输出更稳定。
  • 创意写作：Temperature 0.8~1.0，表达更多样。
  • 总结提炼：Temperature 0.2~0.4，减少胡扯概率。

6.1 按你的硬件给推荐（Windows10 + Ryzen 9700X + RTX5070 12GB）

你的硬件属于本地推理的中高配，优先建议走 GPU 推理路线。

推荐模型组合（按优先级）

场景推荐量化建议说明
C++/工程问答 Coder Instruct 7B~14B Q4_K_M / Q5_K_M 代码相关问题明显优于通用聊天模型
日常对话 通用 Instruct 7B~9B Q4_K_M 速度快、显存压力小
高质量但可接受慢一些 14B Instruct/Coder Q4_K_M 12GB 显存通常可尝试，但上下文别开太大

你这台机器的参数起步值（针对 C++）

temperature: 0.15 ~ 0.35
top_p: 0.85 ~ 0.95
max_tokens: 900 ~ 1600
context_length: 4096 (稳定后可升到 8192)
repeat_penalty: 1.05 ~ 1.12
如果你主要问 C++，优先用 Coder Instruct 模型，不要只用通用聊天模型。
同一个问题可让模型“输出：结论 + 代码 + 边界条件 + 复杂度”，答案会更工程化。

7. 打开本地 API（OpenAI 兼容）

7.1 在 LM Studio 内启动服务

  1. 进入 Developer / Local Server 页面。
  2. 选择已加载模型作为默认模型。
  3. 点击 Start Server。
  4. 确认监听地址（通常：http://127.0.0.1:1234）。

7.2 PowerShell 调用示例（推荐写法，直接打印答案）

$body = @{
  model = "qwen/qwen3.5-9b"
  messages = @(
    @{ role = "system"; content = "你是一个资深C++工程师。回答时先给结论，再给关键代码，最后给注意事项。" }
    @{ role = "user"; content = "解释protobuf和gRPC关系，并给最小C++调用流程" }
  )
  temperature = 0.2
  max_tokens = 1200
  stream = $false
} | ConvertTo-Json -Depth 8

$resp = Invoke-RestMethod `
  -Uri "http://127.0.0.1:1234/v1/chat/completions" `
  -Method Post `
  -ContentType "application/json" `
  -Body $body

# 关键：直接取 content，不要只看对象摘要
$msg = $resp.choices[0].message

if ($msg.content -and $msg.content.Trim().Length -gt 0) {
  $msg.content
} elseif ($msg.reasoning_content) {
  "[提示] 只返回了 reasoning_content，没有最终答案："
  $msg.reasoning_content
} else {
  "[调试] 未取到正文，打印完整返回："
  $resp | ConvertTo-Json -Depth 20
}

"finish_reason = $($resp.choices[0].finish_reason)"
你的截图里出现 finish_reason=length，说明被 max_tokens 截断了。把 max_tokens 提高到 800~1600 更稳。

7.3 Python (OpenAI SDK) 调用示例

from openai import OpenAI

client = OpenAI(
    base_url="http://127.0.0.1:1234/v1",
    api_key="lm-studio"
)

resp = client.chat.completions.create(
    model="your-loaded-model",
    messages=[
        {"role": "system", "content": "你是一个简洁技术助手"},
        {"role": "user", "content": "给我一个CMake最小示例"}
    ],
    temperature=0.4
)

print(resp.choices[0].message.content)
只要你的工具支持 OpenAI 兼容接口，就可以把 base URL 改成本地地址来接 LM Studio。

8. 性能优化（提速最有效的做法）

  1. 先把模型降一级（比如从 14B 降到 7B）。
  2. 量化改到 Q4 再试。
  3. 降低 context length（8k 降到 4k）。
  4. 尽量使用 GPU 加速（如果可选）。
  5. 关闭其他吃内存的大应用（浏览器、IDE 多窗口、虚拟机）。

判断是否“配置过重”

  • 加载模型特别慢或直接失败。
  • 对话首 token 出得很慢（比如超过 10~20 秒）。
  • 系统明显卡顿、频繁占满内存/页面文件。

9. 常见错误排查

9.1 端口被占用（API 启动失败）

现象：

启动本地服务时报端口冲突（常见 1234）。

处理：

netstat -ano | findstr :1234
taskkill /PID <占用进程PID> /F

9.2 模型下载成功但加载失败

原因常见：

  • 模型过大，内存/显存不够。
  • 下载文件损坏或不完整。

处理：

  • 换同模型更小量化（Q4）。
  • 重新下载模型文件。
  • 先用小模型确认软件链路没问题。

9.3 输出特别慢

  • 降低模型规模与上下文长度。
  • 降低最大输出 token。
  • 确认是否真的启用了 GPU 加速。

9.4 中文在终端乱码（PowerShell）

chcp 65001

并尽量使用 UTF-8 文件与终端。

9.5 API 调用 404 / 401 / 500

  • 404：URL 路径写错（确认是 /v1/chat/completions）。
  • 401：部分工具强制校验 API Key，填任意字符串即可（例如 lm-studio）。
  • 500：模型未加载或服务端内部错误，先在 LM Studio 界面确认模型状态。

9.6 返回看起来“没答案”，只看到 finish_reason=length

现象：

  • PowerShell 里只看到 choices : @{index=0; message=; ...}
  • finish_reason=length

根因：

  • 对象摘要输出没有展开 message.content。
  • max_tokens 太小导致被截断。

处理：

$resp.choices[0].message.content
$resp.choices[0].finish_reason
$resp | ConvertTo-Json -Depth 20
  • 把 max_tokens 提高到 800~1600。
  • 针对 C++ 问题，把 system prompt 固定为“资深C++工程师”。
  • 优先使用 Coder Instruct 模型。

10. OpenClaw 如何关闭运行、会不会开机自动运行

10.1 先判断当前是“手动运行”还是“系统服务运行”

openclaw.cmd gateway status
  • 如果看到 Service: Scheduled Task (missing)：说明没有安装为系统服务，默认不会开机自启。
  • 如果看到 service 已安装并 running：说明可能会被系统拉起（取决于任务计划配置）。

10.2 关闭 OpenClaw（按启动方式分别处理）

A. 你在当前终端前台运行了 openclaw.cmd gateway run

  • 直接在那个终端按 Ctrl + C 即可停止。

B. 网关在后台运行（你不知道在哪个终端）

netstat -ano | findstr :18789
taskkill /PID <PID> /F

C. 已安装为系统服务（Scheduled Task）

openclaw.cmd gateway stop

10.3 开机自启动管理

安装系统服务（需要管理员 PowerShell）：

openclaw.cmd gateway install
openclaw.cmd gateway start
openclaw.cmd gateway status

取消自启动（卸载服务）：

openclaw.cmd gateway stop
openclaw.cmd gateway uninstall

10.4 你当前场景的结论（可对照）

  • 你之前日志里出现了 Gateway service missing，这代表没装服务，不会开机自动运行。
  • 但端口 18789 被占用，说明当时有一个手动启动的 Node 网关进程在跑。

11. 进阶玩法

  • 为不同任务准备不同模型预设（代码、总结、翻译、写作）。
  • 把 LM Studio API 接到你自己的工具链（脚本、编辑器插件、Agent）。
  • 做“私有知识库问答”：先做文档切片，再把检索结果喂给本地模型。
  • 在离线环境里跑安全审阅场景（不出网数据）。
